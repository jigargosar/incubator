# Incubator

Ideas enter half-formed, get fleshed out with specs/mocks/spikes, and graduate to their own repo when ready.

## Structure

- Flat — no nested folders
- Markdown files only
- One file per idea

## Lifecycle

1. Jot down the idea
2. Flesh out with specs, mocks, research
3. Spike if needed to validate feasibility
4. Graduate to its own repo when ready

## Spikes

- Keep minimal — just enough to validate the idea
- Spike code lives alongside the idea's markdown file
- Delete spike code after the idea graduates

## What this is NOT

- Not an implementation repo — no production code
- Not a blog — no publishing pipeline
- Not a monorepo — ideas graduate out, they don't grow here
